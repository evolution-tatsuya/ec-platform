# A-004 イベント・チケット管理ページ E2Eテスト仕様書

## テスト対象ページ
- **ページ名**: イベント・チケット管理ページ (AdminEventsPage)
- **ページルート**: `/admin/events`
- **権限**: 管理者のみ
- **使用レイアウト**: MainLayout（管理画面用）

## テスト環境
- **前提条件（全テストケース共通）**:
  - 管理者としてログイン済み（`admin@ec-platform.local` / `TestAdmin2025!`）
  - テストデータが投入済み（イベント3件、参加者3名以上）
  - ブラウザ: Chrome最新版
  - 画面解像度: 1920x1080

---

## テストケース一覧

### 1. 基本機能テスト

#### TC-001: ページ表示確認
**テスト目的**: 管理者ログイン後、イベント・チケット管理ページが正常に表示されることを確認

**前提条件**:
- 管理者としてログイン済み

**テスト手順**:
1. サイドバーの「イベント・チケット管理」メニューをクリック

3. ページタイトル「イベント・チケット管理」が表示されることを確認
4. タブが3つ表示されることを確認:
   - フォーム作成
   - 参加者リスト
   - QR受付
5. デフォルトで「フォーム作成」タブが選択されていることを確認

**期待結果**:
- ページが正常に読み込まれる
- ページタイトル「イベント・チケット管理」が表示される
- 3つのタブが表示される（デフォルトで「フォーム作成」タブが選択されている）
- Header コンポーネントが表示される

#### TC-022: チェックイン実行
**テスト目的**: チケットコードを入力してチェックインが正常に実行されることを確認

**前提条件**:
- TC-021が成功している
- テストデータに「TICKET-001-ABC」が未使用状態で存在する

4. 「チェックイン」ボタンをクリック

6. ダイアログが閉じることを確認
7. 統計情報が更新されることを確認:
   - 使用済みが1増える
   - 未使用が1減る
8. 「参加者リスト」タブに切り替え
9. 該当チケットのステータスが「使用済み」になっていることを確認

**期待結果**:
- ダイアログが閉じる
- チェックイン完了メッセージが表示される
- 統計情報が更新される
- 参加者リストのステータスが「使用済み」に変わる
- チケットコード入力フィールドがクリアされる

## 注意事項

1. **テストデータの準備**: 各テストケース実行前に必ず適切なテストデータが存在することを確認してください
2. **テスト実行順序**: テストケースは独立して実行できるように設計されていますが、TC-001（ページ表示確認）は必ず最初に実行してください
3. **クリーンアップ**: テスト実行後、追加/編集/削除したデータは必ずクリーンアップしてください
4. **スクリーンショット**: エラー発生時は必ずスクリーンショットを取得してください
5. **API通信ログ**: DevToolsのNetworkタブで全API通信を記録してください
6. **未実装機能**: 現在未実装の機能（CSV出力、Excel出力、詳細表示、二重チェックイン防止等）については、将来的に実装予定のため、テストケースには含めていますが、現時点では実行できません
