# A-002 商品管理ページ E2Eテスト仕様書

## テスト対象ページ
- **ページ名**: 商品管理ページ (AdminProductsPage)
- **ページルート**: `/admin/products`
- **権限**: 管理者のみ
- **使用レイアウト**: MainLayout（管理画面用）

## テスト環境
- **前提条件（全テストケース共通）**:
  - 管理者としてログイン済み（`admin@ec-platform.local` / `TestAdmin2025!`）
  - テストデータが投入済み（商品50点、在庫ログ）
  - ブラウザ: Chrome最新版
  - 画面解像度: 1920x1080

---

## テストケース一覧

### 1. 基本機能テスト

#### TC-001: ページ表示確認
**テスト目的**: 管理者ログイン後、商品管理ページが正常に表示されることを確認

**前提条件**:
- 管理者としてログイン済み

**テスト手順**:
1. サイドバーの「商品管理」メニューをクリック
2. URLが `/admin/products` に遷移したことを確認
3. ページタイトル「商品管理」が表示されることを確認
4. 「新規商品追加」ボタンが表示されることを確認
5. タブ「商品一覧」「在庫管理」が表示されることを確認

**期待結果**:
- ページが正常に読み込まれる
- ページタイトル「商品管理」が表示される
- 「新規商品追加」ボタンが表示される
- タブが2つ表示される（デフォルトで「商品一覧」タブが選択されている）

#### TC-002: 商品一覧の表示確認（DataGrid）
**テスト目的**: 商品一覧がDataGridで正常に表示されることを確認

**前提条件**:

**テスト手順**:

2. DataGridのカラムヘッダーを確認:
   - チェックボックス列
   - 商品名
   - 種別
   - カテゴリー
   - 価格
   - 在庫

   - ステータス

3. 各行に商品データが表示されることを確認
4. ページネーションコントロールが表示されることを確認

**期待結果**:

- 商品データが各行に表示される

- イミュータブルログ方式のため、在庫数 = 入庫ログ合計 - 出庫ログ合計 で計算される

---

## エラーハンドリングテスト

### TC-021: 権限エラー（未ログイン）
**テスト目的**: 未ログイン状態でアクセスするとログインページにリダイレクトされることを確認

**前提条件**:
- ログアウト状態

**テスト手順**:
1. URLバーに `/admin/products` を直接入力してアクセス
2. リダイレクトされることを確認
3. ログインページ（`/admin/login`）に遷移することを確認

**期待結果**:
- 商品管理ページは表示されない
- ログインページにリダイレクトされる
- リダイレクト後のURLパラメータに `redirect=/admin/products` が含まれる（リダイレクト後の遷移先を記憶）

---

### TC-022: ネットワークエラー
**テスト目的**: API呼び出し失敗時にエラーメッセージが表示されることを確認

**前提条件**:
- TC-001が成功している
- ブラウザのDevToolsでネットワークをオフラインに設定可能

**テスト手順**:
1. 商品管理ページを表示
2. DevToolsでネットワークをオフラインに設定
3. 「新規商品追加」で商品を登録しようとする
4. エラーメッセージが表示されることを確認
5. ネットワークをオンラインに戻す

**期待結果**:
- API呼び出しが失敗する
- エラーメッセージ（例: 「通信エラーが発生しました」）が表示される
- ダイアログは開いたまま（データが失われない）

---

## レスポンシブデザインテスト

### TC-023: モバイル表示確認
**テスト目的**: モバイル画面サイズでも基本機能が使用できることを確認

**前提条件**:
- TC-001が成功している

**テスト手順**:
1. ブラウザの表示サイズを375x667（iPhone SE）に変更
2. ページが正常に表示されることを確認
3. サイドバーがハンバーガーメニューに変わることを確認
4. DataGridが横スクロール可能になることを確認
5. 「新規商品追加」ボタンをタップ
6. ダイアログが画面全体に表示されることを確認

**期待結果**:
- モバイルサイズでも基本機能が使用できる
- DataGridは横スクロールで全列を確認できる
- ダイアログは画面全体に表示される（fullScreen）

## パフォーマンステスト

### TC-024: 大量データ表示
**テスト目的**: 商品が1000件以上存在する場合でも快適に動作することを確認

**前提条件**:
- テストデータとして商品1000件以上が登録されている

**テスト手順**:
1. 商品管理ページにアクセス
2. ページ読み込み時間を計測（3秒以内）
3. DataGridが正常に表示されることを確認
4. ページネーション操作がスムーズであることを確認
5. 検索・絞り込み操作がスムーズであることを確認

**期待結果**:
- ページ読み込み時間が3秒以内
- DataGridの操作が遅延なく動作する
- 検索・絞り込みが1秒以内にレスポンスする

**備考**:
- サーバーサイドページネーション実装を推奨
