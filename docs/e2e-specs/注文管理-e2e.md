# A-003 注文管理ページ E2Eテスト仕様書

## テスト対象ページ
- **ページ名**: 注文管理ページ (AdminOrdersPage)
- **ページルート**: `/admin/orders`
- **権限**: 管理者のみ
- **使用レイアウト**: MainLayout（管理画面用）

## テスト環境
- **前提条件（全テストケース共通）**:
  - 管理者としてログイン済み（`admin@ec-platform.local` / `TestAdmin2025!`）
  - テストデータが投入済み（注文20件、顧客10名）
  - ブラウザ: Chrome最新版
  - 画面解像度: 1920x1080

---

## テストデータ

### 必要なテストデータ
```yaml
注文:
  - 入金待ち（PENDING_PAYMENT）: 5件
    - 決済方法: 銀行振込（未入金）
  - 発送準備中（PREPARING）: 5件
    - 決済方法: クレジットカード（決済完了）
  - 発送済み（SHIPPED）: 5件
    - 配送業者: ヤマト運輸、佐川急便、日本郵便
    - 追跡番号: あり
  - 配達完了（DELIVERED）: 3件
  - キャンセル（CANCELLED）: 2件

顧客:
  - 10名（各注文に紐付け）
  - 氏名、メール、電話番号、住所を持つ

注文商品:
  - 各注文に1-4商品を含む
  - 商品名、数量、単価を持つ

配送オプション:
  - 土日着指定あり: 5件
  - 希望配達日設定あり: 8件
  - 時間帯指定あり: 10件

備考:
  - 備考ありの注文: 7件
  - 備考なしの注文: 13件
```

## 注意事項

1. **テストデータの準備**: 各テストケース実行前に必ず適切なテストデータが存在することを確認してください
2. **テスト実行順序**: テストケースは独立して実行できるように設計されていますが、TC-001（ページ表示確認）は必ず最初に実行してください
3. **クリーンアップ**: テスト実行後、追加/編集/削除したデータは必ずクリーンアップしてください
4. **スクリーンショット**: エラー発生時は必ずスクリーンショットを取得してください
5. **API通信ログ**: DevToolsのNetworkタブで全API通信を記録してください
6. **コンソールログ確認**: 現在の実装ではconsole.logで処理確認を行うため、DevToolsのConsoleタブを開いてください
7. **Phase 13での変更点**: API連携実装時に、フィルタリング機能、発送処理のAPI呼び出し、エラーハンドリングが追加される予定です
8. **Phase 14での変更点**: pdfme実装時に、書類生成機能が実際のPDFダウンロードとして動作する予定です
